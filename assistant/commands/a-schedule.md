检查当前作息状态，提醒当前时段任务，检测断更天数。

## 前置检查

先加载配置获取 Vault 路径：

```bash
if [ -f ~/.claude/plugins/config/assistant/settings.sh ]; then
    source ~/.claude/plugins/config/assistant/settings.sh
    echo "Vault: $VAULT_PATH"
else
    echo "ERROR: 未配置。请先运行 /a-setup"
    exit 1
fi
```

如果未配置，停止执行并提示用户运行 `/a-setup`。

以下所有文件路径都以 `$VAULT_PATH` 为前缀。

## 执行流程

### Phase 1: 读取作息配置

日程表：

| 时段 | 任务 | 优先级 |
|------|------|--------|
| 8:30-10:30 | 自媒体 | ⭐⭐⭐ 最重要 |
| 10:30-12:00 | 产品+Marketing | ⭐⭐ |
| 13:00-16:00 | 外包 | ⭐⭐ |
| 21:30-22:30 | 外包补时 | ⭐ |
| 22:30 | 睡觉 | - |

### Phase 2: 判断当前时段

根据当前时间（GMT+8）匹配时段：

```
时间判断逻辑：
- 8:30-10:30 → 自媒体（⭐⭐⭐ 最重要，禁止被外包占用）
- 10:30-12:00 → 产品+Marketing
- 12:00-13:00 → 午餐+休息
- 13:00-16:00 → 外包
- 16:20 → 接娃
- 17:00-21:30 → 家庭+碎片任务
- 21:30-22:30 → 外包补时
- 22:30+ → 应该睡觉了
- 其他 → 非工作时段
```

### Phase 3: 断更检测

1. 扫描 `03-Areas/media/topics/*.md` 中所有文件
2. 提取每个文件 frontmatter 中的 `published:` 字段（格式 YYYY-MM-DD）
3. 找出最新发布日期
4. 计算断更天数 = 今天日期 - 最新发布日期
5. 计算累计惩罚 = max(0, 断更天数 - 1) × 200 元

### Phase 4: 输出结果

**正常状态（断更 < 2天）**：
```
=== 作息状态 ({当前时间} GMT+8) ===

🕐 当前时段：{时段名}
📋 应做任务：{任务描述}
{优先级提示}

📊 发布状态：
- 最近发布：{日期}《{选题名}》
- 断更天数：{N} 天 ✅

💡 提示：专注 {任务}，避免切换！
```

**警告状态（断更 = 1天）**：
```
⚠️ 断更警告：
- 最近发布：{日期}《{选题名}》
- 断更天数：1 天
- 再断更1天触发惩罚！

🔥 建议：今天必须发布一期内容！
```

**惩罚触发状态（断更 >= 2天）**：
```
🚨 惩罚触发！

📊 断更状态：
- 最近发布：{日期}《{选题名}》
- 断更天数：{N} 天
- 累计惩罚：{(N-1) × 200} 元

💸 惩罚执行：
根据惩罚机制（累计制），需要给监督人转 {金额} 元

💡 止损建议：
1. 立即停下手头工作
2. 用最快方式发一期（哪怕是简短内容）
3. 恢复日更节奏
```

**自媒体时段特别提示**（8:30-10:30）：
```
🛡️ 自媒体时段保护中
这是你最重要的创作时间，禁止做外包！
```

**晚间提示**（>= 22:30）：
```
🌙 已过睡觉时间（22:30）
请尽快休息，保证睡眠质量。
```

## 注意事项

- 时间统一使用 GMT+8（北京时间）
- 断更天数从发布日期次日开始计算
- 累计惩罚：超过1天后每天200元

检查当前作息状态，提醒当前时段任务，检测断更天数。

**当前目录就是 Vault**，使用相对路径。

## 执行流程

### Phase 1: 读取作息配置

从 `60-Memory/preferences.md` 读取用户配置的作息：

```markdown
## 作息
- 起床时间: {用户配置}
- 深度工作: {用户配置}
- 午休: {用户配置}
- 结束时间: {用户配置}
```

如果用户未配置作息，提示运行 `/a-setup` 完成配置。

### Phase 2: 判断当前时段

根据当前时间（GMT+8）和用户配置匹配时段：

```
时间判断逻辑（基于用户配置）：
- 起床时间 ~ 深度工作开始 → 早间准备
- 深度工作时段 → 深度工作（⭐⭐⭐ 最重要）
- 午休时段 → 休息
- 下午 → 常规工作
- 结束时间后 → 应该休息了
- 其他 → 非工作时段
```

### Phase 3: 断更检测

1. 扫描 `20-Areas/media/topics/*.md` 中所有文件
2. 提取每个文件 frontmatter 中的 `published:` 字段（格式 YYYY-MM-DD）
3. 找出最新发布日期
4. 计算断更天数 = 今天日期 - 最新发布日期
5. 计算累计惩罚 = max(0, 断更天数 - 1) × 200 元

### Phase 4: 输出结果

**正常状态（断更 < 2天）**：
```
=== 作息状态 ({当前时间} GMT+8) ===

🕐 当前时段：{时段名}
📋 应做任务：{任务描述}
{优先级提示}

📊 发布状态：
- 最近发布：{日期}《{选题名}》
- 断更天数：{N} 天 ✅

💡 提示：专注 {任务}，避免切换！
```

**警告状态（断更 = 1天）**：
```
⚠️ 断更警告：
- 最近发布：{日期}《{选题名}》
- 断更天数：1 天
- 再断更1天触发惩罚！

🔥 建议：今天必须发布一期内容！
```

**惩罚触发状态（断更 >= 2天）**：
```
🚨 惩罚触发！

📊 断更状态：
- 最近发布：{日期}《{选题名}》
- 断更天数：{N} 天
- 累计惩罚：{(N-1) × 200} 元

💸 惩罚执行：
根据惩罚机制（累计制），需要给监督人转 {金额} 元

💡 止损建议：
1. 立即停下手头工作
2. 用最快方式发一期（哪怕是简短内容）
3. 恢复日更节奏
```

**深度工作时段特别提示**（用户配置的深度工作时段）：
```
🛡️ 深度工作时段保护中
这是你最重要的创作时间！
```

**晚间提示**（超过用户配置的结束时间）：
```
🌙 已过结束时间
请尽快休息，保证睡眠质量。
```

## 注意事项

- 时间统一使用 GMT+8（北京时间）
- 断更天数从发布日期次日开始计算
- 累计惩罚：超过1天后每天200元

# 视频创作方法论

核心理念来自 Jenny Hoyos 和 MrBeast 的短视频方法论。

## 核心原则

### 1. 先写两端，再填中间

- **Hook** 决定观众会不会点进来（生死线）
- **末尾句** 决定观众的最终感受（Peak-End Theory）
- **中间内容** 只是连接这两点的桥梁

### 2. Peak-End Theory（峰终效应）

> "人只记得高潮（Peak）和结尾（End）"

- 峰值：视频中最搞笑/最震惊/最有价值的那一刻
- 结尾：最后一句话留下的印象
- Payoff 后**立即结束**，不要拖泥带水

## Hook 设计（0-3秒）

### 三秒法则

Hook 必须在 3 秒内完成：
1. 打断滑动惯性
2. 创造好奇心缺口
3. 承诺价值

### 钩子类型

| 类型 | 核心机制 | 适用场景 |
|------|----------|----------|
| 痛点共鸣型 | 说出用户的痛 | 教程、解决方案 |
| 反认知型 | 挑战常识 | 科普、揭秘 |
| 焦虑放大型 | 制造紧迫感 | 干货、提醒 |
| 利益承诺型 | 直接给好处 | 教程、技巧 |
| 故事悬念型 | 开个头不说完 | 故事、经历 |
| 视觉冲击型 | 画面震撼 | 挑战、展示 |

### Shock/Intrigue/Satisfy 三步法

1. **Shock**：用震惊的事实/画面打断滑动
2. **Intrigue**：制造好奇缺口，让观众想知道"然后呢"
3. **Satisfy**：承诺将给出满意的答案/结果

## 结构框架

### But-So 叙事

```
[陈述] → 但是 [转折] → 所以 [推进] → 但是 [新转折]...
```

每 30-45 秒加入一个"但是"，保持观众的好奇心。

### 常用框架

| 框架 | 适用场景 |
|------|----------|
| AIDA | 产品推广、教程 |
| PASTOR | 问题解决型 |
| PAS | 简短文案 |
| But-So | 故事/教程 |
| Survival Structure | 挑战/竞技 |

## 节奏控制

```
快（Hook）→ 中（展开）→ 快（高潮）→ 中 → 快（结尾）
```

- **快节奏**：抓注意力、制造冲击
- **中节奏**：传递信息、建立背景
- **节奏变化**：保持观众的新鲜感

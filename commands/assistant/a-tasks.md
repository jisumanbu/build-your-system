显示任务概览并提供智能建议。

## 执行步骤

### 0. 时段提示（置顶）

**必须最先执行**，在输出任何内容之前：

1. 获取当前时间（GMT+8）
2. 根据时间判断当前时段：
   - 8:30-10:30 → 自媒体（⭐⭐⭐ 最重要）
   - 10:30-12:00 → 产品+Marketing
   - 12:00-13:00 → 午餐休息
   - 13:00-16:00 → 外包
   - 16:20 → 接娃
   - 17:00-21:30 → 家庭+碎片任务
   - 21:30-22:30 → 外包补时
   - 22:30+ → 应该睡觉了

3. 在概览最开头显示：

```
⏰ {HH:MM} | 当前时段：{时段名}
```

4. **自媒体时段保护**：如果当前是 8:30-10:30，额外显示：

```
🛡️ 自媒体保护时段
这是最重要的创作时间，禁止做外包！
```

5. **晚间提醒**：如果当前时间 >= 22:30，显示：

```
🌙 已过睡觉时间，请尽快休息
```

---

### 1. 读取任务文件

读取以下文件：
- `02-Tasks/inbox.md` - 待处理任务
- `02-Tasks/active.md` - 活跃任务

### 2. 解析任务

识别 Obsidian Tasks 格式的任务（参考 vault-structure skill）：
- `- [ ]` 未完成
- `- [x]` 已完成
- `📅 YYYY-MM-DD` 到期日期
- `⏫` 高优先级 / `🔼` 中优先级 / `🔽` 低优先级

### 3. 生成概览

```
=== 任务概览 ===

📥 Inbox ({数量}个待处理)
{列出 inbox 中的任务，最多显示 5 个}

🎯 今日重点
{列出 active.md 中标记为今日的任务}

⏰ 即将到期
{列出 3 天内到期的任务}

📊 统计
- Inbox: X 个
- 活跃: X 个
- 本周到期: X 个
```

### 4. 智能建议

根据任务状态提供 1-2 条建议：

- 如果 inbox 超过 5 个：建议整理，移动到 active 或 someday
- 如果今日无重点任务：建议从 inbox/active 中选择 1-3 个 MIT
- 如果有逾期任务：提醒处理或调整日期
- 如果任务较少：鼓励用户，询问是否需要规划新任务

### 5. 快捷操作提示

```
---
💡 快捷操作：
- /a-capture <内容> - 添加新任务
- /a-review - 进行每日回顾
```

## 注意事项

- 保持输出简洁，不要太长
- 重点突出今日和即将到期的任务
- 建议要具体可行

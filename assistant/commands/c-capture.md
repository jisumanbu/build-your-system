将内容捕获到统一 inbox，智能打标签。

**$ARGUMENTS**

## 执行流程

### 0. 检查初始化状态

先检查 `60-Memory/profile.md` 是否存在。如果不存在，提示用户先运行 `/a-setup` 完成初始化，然后停止执行。

### 1. 分析内容，识别标签

**类型标签**：使用 capture-rules skill 中的统一规则。

**领域标签**：读取 `60-Memory/tag-mapping.md` 获取用户配置的领域标签和关键词。
- 如果 tag-mapping.md 不存在，使用默认领域标签（work, life, learning）

识别顺序：
1. 类型标签（必选1个）：优先显式标签，其次关键词触发
2. 领域标签（可选，可叠加）：根据 tag-mapping.md 中的关键词识别
3. 状态标签（可选）

### 1.5 间隙记录智能识别（新增）

**触发条件**：内容匹配以下任一模式时，自动识别为 `#pause`：

| 模式 | 示例 |
|------|------|
| `刚完成 xxx，接下来 yyy` | 刚完成文章初稿，接下来准备开会 |
| `完成了 xxx，下一步 yyy` | 完成了报告，下一步是测试 |
| `xxx 做完了，接下来 yyy` | 代码做完了，接下来写文档 |
| 包含"分心"、"又刷了"、"跑偏了" | 又刷了20分钟，该回去工作了 |

**识别为 #pause 后的特殊处理**：

1. **自动提取结构**：
   - "刚完成/完成了/做完了"后面的内容 → 完成项
   - "接下来/下一步"后面的内容 → 下一步

2. **情绪自动识别**（使用 capture-rules skill 中的情绪关键词）：
   - 包含"顺利、不错、搞定" → 😊 顺利
   - 包含"卡住、困难、问题" → 😔 卡住
   - 包含"分心、刷、跑偏" → 📱 分心
   - 无特殊关键词 → 😐 一般

3. **分发到间隙日志区块**：
   - 写入 `00-Inbox/{日期}.md` 的 `## 间隙日志` 区块
   - 使用间隙日志专用格式（见下方）

### 2. 写入目标文件

#### 2a. 普通捕获 → `00-Inbox/capture.md`

**排序规则**：新内容插入到文件已有内容**之前**（倒序，最新在最上面）

使用 Write/Edit 工具将内容插入到 capture.md 顶部（`## 待处理` 之后）：

```markdown
---
### {月}-{日} {时}:{分}
{内容}
{类型标签} {领域标签} {状态标签}
---
```

#### 2b. 间隙记录 → `00-Inbox/{YYYY-MM-DD}.md`

当识别为 `#pause` 时，写入当日日志的 `## 间隙日志` 区块：

1. 读取 `00-Inbox/{YYYY-MM-DD}.md`
2. 如果 `## 间隙日志` 区块不存在，在文件开头创建
3. 在区块内插入新记录（最新在最上面）

```markdown
## 间隙日志

### {HH:MM}
- **完成**: {提取的完成内容}
- **感受**: {emoji} {感受描述}
- **下一步**: {提取的下一步内容}
- **备注**: {原始内容中的其他信息，如果有}

---
```

### 3. 反馈用户

**普通捕获**：
```
已捕获 {类型标签} {领域标签}
```

**间隙记录**：
```
⏱️ {HH:MM} | 间隙记录已保存
感受: {emoji}
下一步: {下一步内容}
```

## 重要规则

1. **所有内容统一进 inbox**，不做分发（间隙记录除外，直接写入当日日志）
2. /o-review 时再决定去向
3. 类型标签必选1个，其他可选
4. 直接保存，不询问确认
5. **当前目录就是 Vault**，使用相对路径
6. **间隙记录优先**：当内容匹配间隙模式时，优先识别为 #pause

## 示例

### 普通捕获

输入：明天要给客户发报价单
→ `#task #outsourcing`
→ 追加到 capture.md
→ 回复：已捕获 #task #outsourcing

输入：可以做期关于创业的视频
→ `#topic #media #可做视频`
→ 追加到 capture.md
→ 回复：已捕获 #topic #media #可做视频

输入：等客户回复报价确认
→ `#waiting #outsourcing`
→ 追加到 capture.md
→ 回复：已捕获 #waiting #outsourcing

### 间隙记录（智能识别）

输入：刚完成文章初稿，感觉不错，接下来准备开会
→ 识别为 `#pause`
→ 提取：完成="文章初稿"，感受=😊顺利，下一步="准备开会"

→ 回复：⏱️ 14:32 | 间隙记录已保存
         感受: 😊 顺利
         下一步: 准备开会

输入：又刷了20分钟Twitter，该回去工作了
→ 识别为 `#pause`
→ 提取：完成="-"，感受=📱分心，下一步="回去工作"
→ 写入间隙日志
→ 回复：⏱️ 10:46 | 间隙记录已保存
         感受: 📱 分心
         下一步: 回去工作
